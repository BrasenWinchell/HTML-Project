Specificity and CSS

Alright, so in HTML you can have attributes called class and id, these are both used for the purposes of organizing specific things into a group so they can be changed/editted by CSS and JavaScript. Classes can be used more than once, ids cannot, and ids are more speicifc than classes, which means that they ids will always override classes when they are changed. Typically, you want to save ids for javaScript. Now lets talk specificity, if you have something inline, it wins every time, typically you want to put that in a <div> or in the head. Back to CSS, in the Lorem above, I have two <div> with classes (class="") and named it para1, then in CSS, I had it make everything with the class para1, change its background color to blue. I repeated the same thing with the class para2 and the color green.

.para1{
    background-color: royalblue;
}

.para2{
    background-color: seagreen;
}
